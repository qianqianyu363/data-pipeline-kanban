# 📊 需求下发登记全链路看板 — 设计文档

## 一、数据范围

| 数据源 | 范围 | 说明 |
|--------|------|------|
| 需求下发登记表 | 需求下发→采集→审核→闭环 | 你维护，Excel/企微文档 |
| 结算明细 | 1-4月费用数据 | 12条记录，含单价和金额 |
| 加工发布 | 外部导入 | 预留板块，等你补充 |

## 二、环节与指标

### 🔴 环节1：需求下发
- 需求发布及时率（是否按时下发）
- 需求量（下发总量）
- 需求初审通过率（下发前质量关）
- 需求问题率（采集后暴露的质量关）

### 🟠 环节2：采集执行（4个Tab）
- **📊 产能**：各环节转化漏斗（收到→下发→采集→反馈→核实）
- **💰 费用执行**：月度预算执行、按类型费用分布（来自结算明细）
- **📈 采集进度**：按结算类型/需求来源的完成率对比
- **📋 任务清单**：每项任务的进度表格（进度条、闭环状态、及时率）

### 🟡 环节3：审核打回
- 打回数量、根因分布、自行更正数、需求准确率

### 🟢 环节4：闭环验证
- 闭环率、全部闭环率

## 三、筛选逻辑

| 筛选维度 | 选项 | 联动范围 |
|---------|------|---------|
| 需求项 | 全部/市场价/采购建议价 | KPI卡片、各环节、预警 |
| 结算类型 | 全部/价格采集/价格更新/全系列报价单 | KPI卡片、各环节、预警 |
| 月份 | 全部/03-06月 | KPI卡片、各环节、预警 |
| **月度趋势** | — | **独立于筛选，始终显示全部月份** |

## 四、文件结构

```
看板/
├── index.html          ← 看板页面（主要代码）
├── data.json           ← 所有数据（tasks + settlements + 配置）
├── 需求下发登记表.xlsx  ← 原始Excel数据源
├── .nojekyll            ← 禁用Jekyll（GitHub Pages需要）
├── README.md            ← 使用说明
└── 看板设计文档.md       ← 本文档
```

## 五、更新数据流程

```
企微文档改数据
  ↓ 导出Excel保存到本地
  ↓ 打开看板页 → 导入(CSV/Excel) → 生成看板
  ↓ 导出 data.json → 上传 GitHub 替换原文件
  ↓ 1分钟后刷新，领导看到最新数据
```

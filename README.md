## 📋 看板模块

| 模块 | 内容 |
|------|------|
| KPI 总览 | 总需求量、采集完成数、及时率等核心指标 |
| 环节1：需求下发 | 需求发布及时率/准确率/覆盖率 vs 目标 |
| 环节2：采集执行 | 各项目采集完成率（条形图+目标线） |
| 环节3：审核打回 | 打回根因分析（饼图） |
| 环节4：上线闭环 | 已闭环/进行中/未启动状态 |
| 全链路健康度 | 六维度雷达图评分 |
| 结算与预算 | 月度结算+累计目标组合图 |
| 关键预警 | P0/P1 问题与建议行动项 |

## 🔄 更新数据（3种方式）

### 方式1：直接编辑 data.json（推荐）
1. 打开 [data.json](data.json) 文件，直接修改数据
2. 提交到 GitHub 仓库
3. 等待 1-2 分钟，刷新看板页面即生效

### 方式2：CSV 导入（从企微文档导出）
1. 在企微文档中导出为 CSV 文件
2. 打开看板页面 → 点击「导入CSV」
3. 检查数据无误 → 点击「应用」
4. 点击「导出 data.json」→ 上传到 GitHub 仓库替换原文件

### 方式3：在线编辑
1. 打开看板页面 → 点击右下角「✏️ 编辑数据」
2. 修改 JSON 数据 → 点击「应用」预览
3. 满意后点「导出 data.json」上传到 GitHub

## 🛠 技术栈

- 纯前端 HTML + CSS + JavaScript
- [ECharts 5](https://echarts.apache.org/) 图表库（柱状图/条形图/饼图/雷达图/组合图）
- [PapaParse](https://www.papaparse.com/) CSV 解析
- 数据与视图分离（data.json）
- 托管于 GitHub Pages（免费 CDN）
